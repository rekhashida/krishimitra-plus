What is React.js?
React.js (usually called React) is a free JavaScript library for building websites and web apps. It was created by Facebook (now Meta) and is one of the most popular tools for making modern user interfaces.

Think of a website as a set of building blocks called components. Each component is a small, reusable piece of UI — like a button, a form, a navigation bar, or a whole page.

Instead of writing one huge HTML file and manually updating every part when something changes, React lets you:

Break the app into pieces — e.g. Logo, BottomNav, FarmerDashboard
Reuse those pieces — the same logo appears on every page without copying code
Update the screen automatically — when a farmer registers or a job is posted, React refreshes only the parts that changed
A simple analogy: React is like LEGO blocks for websites. You build small blocks once, snap them together, and change one block without rebuilding the whole thing.

Why we use React for KrishiMitra+
Here’s why React fits this project well:

1. Many screens, one app
KrishiMitra+ has many pages — Welcome, Farmer Register, Worker Dashboard, Crop Scanner, Find Jobs, etc. React makes it easy to manage all of them and move between pages smoothly.

2. Reusable parts
Things that repeat everywhere — the logo, bottom navigation, forms, cards — are built once and used on every page. That keeps the app consistent and easier to maintain.

3. Interactive features
The app isn’t just static text. Users:

Register as farmers or workers
Scan crops and see results
Post and apply for jobs
Give star ratings
React handles these updates on the screen without reloading the whole page.

4. Mobile-friendly
React works well for responsive layouts, so the app looks good on phones — important for farmers and workers in the field.

5. Connects easily to Supabase
React works smoothly with backends like Supabase for saving farmers, workers, jobs, and ratings in a database.

6. Large community and future growth
React is widely used, well documented, and easy to extend. You can later add things like voice support, offline mode, or a real AI crop scanner without rebuilding from scratch.

In one sentence
React lets us build KrishiMitra+ as a fast, organized, mobile-friendly app where each screen and feature is a reusable piece that updates automatically when users interact with it.

1. What is form validation and why does it matter?
Form validation means checking the user’s input before saving it.

Examples in this form:

Name cannot be blank
Phone must be exactly 10 digits (+91 added automatically)
State and district must be selected
Farm size must be greater than 0
At least one crop must be checked
A profile photo must be uploaded
Why it matters:

Stops bad or incomplete data from reaching the database
Gives clear feedback in Hindi and English so farmers understand what to fix
Reduces wrong phone numbers, missing villages, and empty profiles
Makes the app more trustworthy and easier to use in rural areas
2. How does data save to Supabase?
When Supabase is configured (.env with your URL and key), this happens:

User fills form → Validation passes → Data sent to Supabase
Step-by-step:

Profile photo (if possible) is uploaded to Supabase Storage bucket profile-photos
A public URL for the photo is generated
Farmer details are inserted into the farmers table:
{
  name: "Ramesh Patel",
  phone: "+919876543210",
  village: "Anand",
  district: "Anand",
  state: "Gujarat",
  farm_size: 5,
  crop_types: ["Wheat", "Cotton"],
  profile_photo_url: "https://...supabase.co/.../photo.jpg"
}
Supabase returns the saved row with a unique id
That profile is stored in the app (and localStorage) so the farmer stays logged in
If Supabase is not configured, data is saved locally in the browser so you can still test the app.

Database setup: Run the updated supabase-schema.sql in your Supabase SQL Editor to create the new columns (district, crop_types, profile_photo_url) and the photo storage bucket.

3. What happens when the user clicks Register?
Here is the full flow:

Click "Register"
       ↓
Validate all fields
       ↓
   Any errors? ──Yes──→ Show red messages (EN + HI) under each field. STOP.
       ↓ No
Show spinner: "Registering... / पंजीकरण हो रहा है..."
       ↓
Upload photo to Supabase Storage (if configured)
       ↓
Insert farmer record into Supabase `farmers` table
       ↓
Save profile in app memory + localStorage
       ↓
Mark user as "farmer"
       ↓
Show success screen:
  ✓ "Registration Successful!"
  ✓ "पंजीकरण सफल!"
       ↓
Wait 2.5 seconds
       ↓
Redirect to Farmer Dashboard
If something fails (e.g. network or database error), the spinner stops and a bilingual error message is shown at the top of the form.


Why do we need availability dates?
Agricultural work is seasonal and time-bound. Farmers need to know when a worker is free, not just who they are.

Without dates:

A farmer might call a worker who is already busy harvesting elsewhere.
Workers get irrelevant job offers.
Both sides waste time and phone calls.
With dates, KrishiMitra+ can:

Show farmers only workers available during their job period
Match harvest jobs to workers free in that window
Plan sowing/spraying work around worker schedules
Filter out workers whose availability has expired
Example:
A wheat harvest job runs 15 June – 30 June. Only workers with availability covering those dates should appear — that is what these two date fields enable.
